计算机网络五层模型入门
1.物理层
  字面意思，物理连接两个计算机，比如物理层指的是网线，光纤等，允许计算机之前能用0、1进行传递的电信号。
2.数据链路层
  一种传输规范，让计算机之间传输的电信号规范起来，要不光是0、1，哪个计算也不知道代表什么意思
  2.1 以太网协议
    以太网协议规定了计算机的传输规则，几组电信号定义为一帧，每个帧由head和data组成，并规则head和data的总大小限制
    并规定head的大小为18字节，这样计算机就能指定识别帧并拿到帧中head的内容来进行下一步判断
  2.2 MAC地址
    计算机总要有一个地址吧，要不怎么知道发给谁，连入网络的每一个计算机都会有网卡接口，每一个网卡都会有一个唯一的地址，这个地址就叫做 MAC 地址。
    计算机之间的数据传送，就是通过 MAC 地址来唯一寻找、传送的。MAC地址 由 48 位二进制数所构成，在网卡生产时就被唯一标识了。
  2.3 广播与ARP协议
    2.3.1 广播
     在同一个子网中，计算机通过广播的形式进行，就是说发送者会将数据发送给子网中所有跟发送者相连的，发送的帧中会有接受者的mac地址，
     每个子网中计算机会接收到这个数据包，然后将head中mac地址取出来跟自己的mac地址进行比较，如果相同则接受，不同则丢弃这个数据包。
    2.3.2 ARP协议
     发送者如何知道接受者的mac地址的，通过ARP协议，这个跟ip地址有关，等会再回来写

3.网络层
   每次都是通过广播发送出去，全球每个计算机都会接受其他所有发送者的数据再进行筛选，计算机会疯掉，所以出现了子网，也就是说
   能通过广播的地址都是在同一个子网下的，不是一个子网的，转发给网关，让网关进行转发，那怎么才算是一个子网下呢，通过ip协议，
  3.1 ip协议
   目前通用的是ipv4，也就是32位二进制数组成，比如192.168.1.2
   而这个ip地址由网络部分和主机部分组成，具体的组成方式不固定，有可能是网络部分24位，主机8位或者网络16位，主机16位，那怎么来确定呢，
   通过子网掩码，子网掩码每8位只能是都是1或者都是0，也就是255或者0，这样在ip地址和子网掩码进行与的位运算，会把主机部分消除掉，通过对比网络部分
   相同则认为两个ip地址是处在同一个子网下
  3.2 ARP协议
   知道了ip地址和子网掩码就可以知道两台计算机是不是在同一个子网下，如果在的话，那么通过广播形式给其他计算机，其他计算机在拿到这个数据包后确认是自己的地址后
   再讲自己的mac地址回传给发送者，则发送者就知道了接受者的mac地址，可是发送请求mac地址和传送数据都是同一个数据包，接受者怎么知道是不一样的，在请求mac地址的
   head中不仅存放接受者ip地址，其中接受者mac地址是特殊的mac地址。
   如果不是同个子网下的，则发送给网关，通过网关进行转发
  3.3 DNS服务器
  帮助我们将域名解析成ip地址的

  网络层的功能就是让我们在茫茫人海中，能够找到另一台计算机在哪里，是否属于同一个子网
4.传输层
  网络层协议是主机跟主机之间的连接协议，传输层协议这是端口和端口之间的协议，最常见的就是TCP协议和UDP协议
